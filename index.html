##iOS中设置圆角

在view中设置圆角和uiimageview中设置圆角有点不一样

当没有用到性能调优的时候就感觉离屏渲染就是把圆角画出来,现在真的用到了,认真的看了几种方法,网上的文章都是列出几种方法,然后说自己的方法是最厉害的,别的方法不是内存占用就是离屏渲染,踩过了就记录一下.

1,普通的

这个 离屏渲染 ,切圆角多了就会掉帧,但是是直接改图片框的,无论图片大小都没有关系

2, CAShapeLayer和UIBezierPath

这个离屏渲染,是直接覆盖图片框的,无论图片大小都没有关系 ,对内存的消耗最少,渲染快速

3, Core Graphics和UIBezierPath

从图片上下文裁剪,这个方法是目前我找到的比较好的方法,没有离屏渲染,但是,但是这个裁剪的圆角是根据图片的大小
,如果图片不一样大小裁剪出来的圆角呈现的也是不一样的,所以用之前最好确定图片的大小,当然裁圆形是没有问题的

  • CGContextTranslateCTM(context, 0, self.size.height);
    CGContextScaleCTM(context, 1.0, -1.0); *
    如果不调用这两个,裁剪出来的图片就是反的

4, DrawRect
网上帖子都说drawrect是最好不要调用的,但是他们的方法中都用到了这个方法,
drawrect的用法是没有离屏渲染的,但是内存会暴涨,所以不是好的方法

5, view上添加一个遮罩
这个方法我是参考的一个大神的XWAddForRoundedCorner
这个方法对性能和流畅性非常好,但是像作者说的,需要背景颜色

多行label富文本的省略号

发表于 2017-06-21 |

多行label下富文本的省略号

多行label下设置行数为3numberOfLines=3,当文本内容超过3的时候,默认text是有一个省略号来表示折叠内容的,
当文本设置为富文本的时候isAttributedContent=yes,后面是没有省略号的,
如果要设置富文本的段落样式,则富文本需要用到NSMutableParagraphStyle,这个style中有这样的一个属性:lineBreakMode
lineBreakMode在text中默认是切断的,但是在富文本中是没有设置的,这时候我们只要将lineBreakMode设置成NSLineBreakByTruncatingTail就乐意在富文本有折叠内容的时候出现省略号.
label.lineBreakMode = NSLineBreakByCharWrapping;以字符为显示单位显示，后面部分省略不显示。
label.lineBreakMode = NSLineBreakByClipping;剪切与文本宽度相同的内容长度，后半部分被删除。
label.lineBreakMode = NSLineBreakByTruncatingHead;前面部分文字以……方式省略，显示尾部文字内容。
label.lineBreakMode = NSLineBreakByTruncatingMiddle;中间的内容以……方式省略，显示头尾的文字内容。
label.lineBreakMode = NSLineBreakByTruncatingTail;结尾部分的内容以……方式省略，显示头的文字内容。
label.lineBreakMode = NSLineBreakByWordWrapping;以单词为显示单位显示，后面部分省略不显示。

自适应行高笔记

发表于 2017-05-07 |

自适应行高笔记

以前的cell不同文本的高度都是用的SDAutoLayout,但是第三方用的时间久了总是想着能够自己钻一下,有一个缺点就是SDAuto在cell自适应高度的时候会多次的去调用setmodel这个方法

​ 多次调用setmodel这个方法,感觉不算什么大问题,但是setmodel 中有许多的业务在这里面,调用的次数多了就显得多余和臃肿,所有想着试一试Masonry和YYKit这两种的布局方式.

​ 首先YYKit,YYKit确实是非常厉害的三方库,对作者也是膜拜的心情,各种富文本布局是非常炫酷也非常实用,但是我觉得yykit的布局是不太明确的,如 uilabel.top 这种写法不能让我直观的感受到label的top到底是哪个,基于这个我就又去看了Masonry

​ 在Masonry中,布局是非常舒服的,相信大多数开发也是实用的Masonry布局,然而听组长说Masonry的布局是没有自使用行高的,但是可以计算label文本内容的高度

想法

布局是习惯了masonry的,所有用masonry做基础布局,在布局过程中遇到了一些问题

  • masonry在setmodel中布局,因为是消息类,所以有一个父类的cell,具体的例如文本,图片,音频继承自这个cell,这就是所谓的工厂模式吧
  • 在父cell中写了一些共有的控件,例如时间、头像,在父cell中的setmodel的时候定义了这些控件的布局,但是在子类中[super setmodel]后,控件还是没有frame,是因为cell的layout还没有刷新frame, 调用[setlayoutifneed] 就可以刷新控件的frame,从而拿到要做cell的高度的底部控件的高
  • 将控件的高保存在模型中(model),代理的控制器中cell高度的代理方法调用数组中对应的cell的model,获取保存在其中的高就可以知道当前cell的高度
  • 但是,这样的高度只有在cell展示出来的时候才可以计算cell的高度,在聊天的类中,是需要根据cell的高度来决定聊天列表向下滚到的底部的距离的,但是新进来的消息是在没有展示cell的时候就执行代理中返回高度的代理方法的,这时候cell的高度为0,列表不会向下滚动,就看不到新的消息,一个简单的解决方法是在消息的字典转model时给model预设一个高度,这样就能保证聊天列表会向下滚动,而且不会超过cell的高度,因为在滚动时会去计算cell的高度,但是预设的高度小于cell的应有高度,则不会滚动到底部. 现在能想到的比较好的方法是yykit的方法,Label这些文本其实是一个块,计算出这个文本块的高度但是是在一个高于model的类layout中,这个layout类实现了计算Label文本块大小的方法,对富文本的支持特别厉害,这样新消息不用cell展示就能知道行高,因为行高保存在了layout类中,同理,我在model中写了这样一个计算高度的方法
  • 表情 表情是在model中计算高度的方法中根据正则表达式找出表情替换,注意是从后向前替换的
  • 图片 自己发送的和文字一样,在model中计算图片的宽高,是保存在缓存中的,所以可以马上提取到图片的大小,如果是对面发送过来的图片,可以先用一个占位图,然后在图片加载完成后计算宽高替换占位图刷新cell高度,这里本来是用的一个第三方,但是这个第三方有一些滑动的bug,现在改用了kvo来监听在sdwebimage加载完成的回调中刷新cell
  • 语音 语音在model中计算语音的长度
  • 电话号码 电话号码应该是带下划线的富文本,这里和表情一样在富文本中根据正则表达式找出符合的电话号码,采用对应的富文本和位置替换
